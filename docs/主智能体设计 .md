## 职业规划智能体设计

目前一共设计了学生能力建模模块（将简历通过 LLM 转化为系统可计算的结构化能力向量 ），就业岗位画像体系构建（基于大语言模型构建岗位画像体系）和匹配引擎（基于大模型），这三个模块都使用了大模型，我们可以分别称它们为：学生能力建模专家，岗位画像专家，人岗匹配专家。

接下来可以构建一个主智能体，用于和用户对话，智能体可以根据任务需求调用这三个专家。

基于 LangChain 和 LangGraph 构建多智能体（Multi-Agent）系统，是优雅的方案。LangGraph 的核心思想是将工作流抽象为**状态图（StateGraph）**，通过维护一个全局的“状态（State）”，让各个智能体（节点）在状态的驱动下协同工作。

为了实现你提出的“主智能体 + 三个专家”的架构，我们需要将之前设计的独立模块封装为 LangGraph 中的节点（Nodes），并由主智能体负责路由（Routing）。


### 二、 模块职责与依赖关系（节点划分）

在状态图中，我们需要定义以下四类节点：

#### 1. 主智能体节点 (Main Agent / Supervisor)

-   **职责**：负责与用户直接交互，意图识别，以及任务分发。它不执行具体的计算或抽取任务。
    
-   **依赖**：依赖大模型（如 Qwen 或 DeepSeek）理解 `messages` 中的用户意图，并通过工具调用（Function Calling）或结构化输出，决定更新 `AgentState` 中的 `next_step`（例如赋值为 `call_student_expert`, `call_job_expert`, `call_match_expert` 或 `reply_to_user`）。
    

#### 2. 学生能力建模专家节点 (Student Expert)

-   **职责**：当主智能体发现用户提交了简历文本，但 `student_profile` 为空或需要更新时被唤醒。
    
-   **行为**：读取 `AgentState['resume_text']`，执行之前设计的简历解析 Prompt，将非结构化文本转换为标准化能力向量 。
    
-   **输出**：将解析结果写入 `AgentState['student_profile']`，并将控制权交还主智能体。
    

#### 3. 岗位画像专家节点 (Job Expert)

-   **职责**：当用户询问特定岗位，或准备进行匹配但缺少目标岗位数据时被唤醒。
    
-   **行为**：根据 `AgentState['target_job_name']` 去本地数据库检索岗位数据，并通过大模型抽取生成结构化画像 。

**注：** 数据库中已经有构建好的岗位画像，如果数据库中有需要检索的岗位画像则不需要调用大模型了。
    
-   **输出**：将结果写入 `AgentState['job_profile']`，并将控制权交还主智能体。
    

#### 4. 人岗匹配专家节点 (Match Expert)

-   **职责**：当 `student_profile` 和 `job_profile` 都已具备，且用户请求匹配时被唤醒。
    
-   **行为**：执行我们之前设计的 LLM-as-a-Judge 逻辑，计算匹配度并输出各维度的得分与差距分析列表 。
    
-   **输出**：将分析报告写入 `AgentState['match_result']`，并将控制权交还主智能体。
    

### 三、 运行图谱（Graph Topology）与调用流

依赖关系和流转逻辑通过 LangGraph 的有向边（Edges）和条件边（Conditional Edges）来绑定。

-   **起点 (START)** -> `Main_Agent`
    
-   **条件路由 (Conditional Edges)**：
    
    -   If `next_step == 'call_student_expert'` -> 跳转到 `Student_Expert`
        
    -   If `next_step == 'call_job_expert'` -> 跳转到 `Job_Expert`
        
    -   If `next_step == 'call_match_expert'` -> 跳转到 `Match_Expert`
        
    -   If `next_step == 'reply_to_user'` -> **终点 (END)** （将系统回复展示给用户）
        
-   **回流 (Normal Edges)**：
    
    -   `Student_Expert` -> `Main_Agent`
        
    -   `Job_Expert` -> `Main_Agent`
        
    -   `Match_Expert` -> `Main_Agent`
        

_(注：专家执行完毕后必须回到主智能体，由主智能体将结构化数据转化为温柔、通俗的自然语言向用户汇报。)_
